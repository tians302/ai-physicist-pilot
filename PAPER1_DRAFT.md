## Abstract (draft)

LLM-driven simulation agents now automate materials workflows, but their
*claims* are typically self-judged. We present a closed-loop AI scientist
whose untrusted proposer (an LLM) can only select from a registry of
typed, versioned capabilities; whose plans are schema-validated before
execution; and whose results pass hard, pre-registered validity gates
before any narration. On a six-capability battery spanning four property
families and two engines, the harness recovers registered answer keys
while rejecting 100% of planted unsupported hypotheses (rejection
precision/recall = 1.0), detecting 26/26 planted corruptions with the
intended gate and zero collateral failures, and — in gate-ablation pilots
— reducing the unflagged-false-claim rate from 0.83 [0.44, 0.97] to
0.00 [0.00, 0.39] (Wilson 95%, **PILOT** N = 6; pre-registered N = 30
pending). Cross-model sensitivity runs expose strongly model-conditional
observables that no per-run gate can catch (C44 spread 138% between SW
and Tersoff-1988 silicon), delimiting honestly what gating can and cannot
guarantee.

## 1. Introduction

Automation of simulation workflows is largely solved; *trustworthiness of
autonomous claims* is not. Existing agents self-judge and/or execute
generated code; reliability is asserted, not measured. Contributions:
(i) a trusted control plane — registry, typed plans, allowlisted engines,
fixed analyzers, pre-registered gates — around an untrusted LLM proposer;
(ii) unsupported-hypothesis rejection as a *measured* behavior;
(iii) headline gate-ablation statistics on false-claim rates;
(iv) a documented taxonomy of what gates cannot catch.

## 3. Framework

Trust boundary (untrusted proposer / trusted control plane / fallible
instruments). 12-stage fixed state machine (`core/loop2.py`). Capability
registry with dimensional validity domains and model applicability cards;
deterministic, fail-closed router — preference can select among
validator-approved candidates, never expand them. Typed plans
(pydantic, `extra="forbid"`); engines produce checksummed `RawRun`s;
fixed versioned analyzers produce unit-checked `ObservationBundle`s;
gates run pre-narration; harness-only scoring; qualitative keys verified,
quantitative keys refused until human-verified provenance.

## 4. Benchmarks and metrics

Property ladder: Callaway κ(T) (thermal-analytic), EOS + elasticity
(structural/mechanical), NPT expansion (thermal), LJ diffusion + GK κ
(transport). Answer-key independence rules; same-model values are
regression fixtures only. Pre-registration protocol
(`PREREGISTRATION_DRAFT.md`): frozen thresholds, N = 30, Wilson CIs,
no exclusions.

## 6. Discussion

What gates cannot catch, with two measured exhibits: (a) the affine-shear
C44 = 109.8 GPa for SW-Si — linear, symmetric, stable, plausible, and
wrong for the physical question (internal-ion relaxation changes it to
56.45); (b) Tersoff-1988's soft C44 — every gate green, 138% cross-model
spread. Hence: gates bound evidentiary validity; protocol review and
cross-model spread bound physical adequacy; claims are model-conditional
by construction. Scope boundary and the path to application (glassy
carbon, Paper 2).
